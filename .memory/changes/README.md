# Изменения проекта

Папка `.memory/changes/` хранит рабочие артефакты значимых изменений в облегченном стиле `OpenSpec`.

Для нового проекта стартовый комплект change-артефактов рекомендуется создавать через `/prj-init`.

Когда использовать:
- задача идет больше одной сессии;
- меняется поведение, интерфейс, интеграция, API или данные;
- есть несколько шагов и важно не потерять контекст решения;
- изменение затрагивает несколько файлов, модулей или областей.

Когда не использовать:
- мелкая локальная правка;
- исправление опечатки или форматирования;
- разовая консультация без изменения проекта.

Рекомендуемая структура:
- `.memory/changes/<change-id>/proposal.md`
- `.memory/changes/<change-id>/design.md`
- `.memory/changes/<change-id>/tasks.md`
- `.memory/changes/<change-id>/spec-delta.md`

Для `/prj-init` используй полный комплект, даже если артефакты пока короткие.

Минимальный набор:
- `proposal.md`
- `tasks.md`

`design.md` и `spec-delta.md` добавляй только когда они действительно нужны.

Правило именования `change-id`:
- `YYYY-MM-DD-short-slug`
- пример: `2026-03-15-auth-refactor`

Роли файлов:
- `proposal.md` — зачем нужно изменение, цель, ожидаемый эффект, границы.
- `design.md` — технический подход, риски, точки расширения, затронутые объекты.
- `tasks.md` — чек-лист и статус выполнения.
- `spec-delta.md` — какие сценарии и требования меняются.

Во всех значимых change-планах финальный блок должен содержать автотесты, исправление ошибок и условие передачи пользователю только после зеленого прогона или явной фиксации блокера.
