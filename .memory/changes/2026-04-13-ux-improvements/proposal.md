# UX Improvements — Proposal

## Зачем

Приложение Trusted Gateway функционально зрелое: запросы, маскировка, анализ, execute_code, suggest_fields. Но пользовательский опыт можно значительно улучшить — ускорить типичные операции, снизить порог входа для новых пользователей и добавить удобства для опытных.

## Цель

Набор независимых UX-улучшений, каждое из которых реализуется отдельной сессией. Ни одно изменение не ломает существующую функциональность — только добавления и улучшения.

## Ожидаемый эффект

- Быстрее работа с запросами (горячие клавиши, история, автодополнение)
- Понятнее первый запуск (онбординг)
- Удобнее управление данными (контекстное меню, экспорт, ресайз)
- Лучше обратная связь (прогресс, звук, toast-уведомления)

## Границы

- Изменения только в `ui.html` и, при необходимости, в `web.go` / `mcp_server.go`
- Не меняем протокол MCP, структуру маскировки, логику NER
- Не добавляем внешних зависимостей (npm, CDN-библиотеки), кроме уже используемого Ace Editor
- Каждый пункт — самостоятельный, может быть реализован и протестирован отдельно
