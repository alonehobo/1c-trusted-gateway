# UX Improvements — Design

## Архитектура изменений

Все улучшения — frontend-only (ui.html), за исключением пунктов, требующих серверной поддержки (история запросов с персистенцией, автодополнение из метаданных). Ниже — технический подход по каждому пункту.

---

## P1 — Высокий приоритет

### 1. Онбординг / Welcome Screen

**Подход:** Overlay-div поверх `#app`, показывается когда `!s.connection.verified && !localStorage.getItem('onboarding_done')`. Три шага: URL → ключ → проверка. После успешного connect — скрывается и ставит флаг в localStorage.

**Файлы:** `ui.html` (HTML + CSS + JS)

**Риски:** Минимальные. Overlay не влияет на существующий UI.

---

### 2. Горячие клавиши

**Подход:** Глобальный `keydown`-handler на `document`. Ace Editor уже обрабатывает свои события — нужно ловить `Ctrl+Enter` внутри редактора через Ace API (`editor.commands.addCommand`).

**Карта клавиш:**
- `Ctrl+Enter` — выполнить запрос (вызывает `doQuery('masked')` или `doApproveCode()` в зависимости от режима)
- `Escape` — отмена запроса (`doCancelQuery()`)
- `Ctrl+S` — сохранение настроек (если открыта вкладка Settings)
- `Ctrl+Shift+H` — открыть историю запросов (п.3)

**Файлы:** `ui.html` (JS — секция инициализации Ace + глобальный listener)

**Риски:** Конфликт с браузерными шорткатами. `Ctrl+S` нужно перехватывать через `e.preventDefault()`. В WebView (Wails) конфликтов меньше, чем в обычном браузере.

---

### 3. История запросов

**Подход:** Хранение в `localStorage` массива `{query, timestamp, rowCount}`, лимит — 50 записей (FIFO). UI — выпадающий dropdown под кнопкой "История" рядом с редактором. Клик по записи — вставка в редактор.

**Структура данных:**
```json
[
  {"query": "ВЫБРАТЬ ...", "ts": 1713020400, "rows": 42, "task": "анализ продаж"},
  ...
]
```

**Файлы:** `ui.html` (HTML dropdown + JS localStorage logic). Сохранение происходит после успешного выполнения запроса.

**Риски:** localStorage ограничен ~5MB — при 50 записях это не проблема.

---

### 4. Обратная связь при действиях с тегами полей

**Подход:** CSS-анимация `.tag-flash` (кратковременная подсветка) + toast "Поле X добавлено в белый список" / "Поле X зашифровано".

**Файлы:** `ui.html` (CSS keyframe + JS в `allowField()` / `reMaskField()`)

**Риски:** Нет.

---

## P2 — Средний приоритет

### 5. Автодополнение в Ace Editor

**Подход:** Ace имеет встроенный completer (`ace/ext/language_tools`). Нужен кастомный completer, который:
- Статически знает ключевые слова языка запросов 1С (ВЫБРАТЬ, ИЗ, ГДЕ, СГРУППИРОВАТЬ, УПОРЯДОЧИТЬ, СОЕДИНЕНИЕ, ДАТАВРЕМЯ, СУММА, КОЛИЧЕСТВО, и т.д.)
- Динамически подгружает имена таблиц/полей из метаданных после подключения (новый API endpoint `/api/metadata_hints` на сервере)

**Файлы:** `ui.html` (JS completer), `web.go` (новый endpoint, опционально)

**Риски:** Без серверной части — только статические ключевые слова. С серверной — нужен запрос метаданных к 1С, может быть медленным.

**Этапы:**
1. Статические ключевые слова (frontend only) — быстро
2. Динамические метаданные (server + frontend) — отдельная задача

---

### 6. Ресайз панелей (draggable splitter)

**Подход:** CSS-элемент `.splitter` (ширина 4px, cursor: col-resize) между двумя `.split-pane`. JS-обработчик `mousedown` → `mousemove` → `mouseup` меняет `flex-basis` левой панели. Сохранение пропорции в localStorage.

**Файлы:** `ui.html` (HTML + CSS + JS)

**Риски:** Нужно корректно обрабатывать min-width обеих панелей. Ace Editor требует `editor.resize()` после изменения размера контейнера.

---

### 7. Экспорт результатов в CSV

**Подход:** Кнопка "CSV" рядом с "Копировать все". Генерация CSV в JS (с учётом замаскированных данных — экспортируются замаскированные значения). Download через `Blob` + `URL.createObjectURL` + `<a download>`.

**Файлы:** `ui.html` (JS)

**Риски:** Большие датасеты (>10k строк) могут медленно генерироваться в браузере. Разумный лимит уже есть на сервере.

---

### 8. Контекстное меню на таблице

**Подход:** Перехват `contextmenu` на `#result-table tbody`. Показ custom-div с пунктами:
- Копировать значение
- Фильтровать по значению (заполняет фильтр)
- Открыть/зашифровать столбец

**Файлы:** `ui.html` (HTML + CSS + JS)

**Риски:** На мобильных устройствах контекстное меню работает иначе, но приложение десктопное.

---

### 9. Elapsed time при выполнении запроса

**Подход:** При старте запроса запоминать `Date.now()`. `setInterval(1000)` обновляет текст в badge "Выполнение... (3с)". Останавливается при смене `query_state`.

**Файлы:** `ui.html` (JS в `render()` и `doQuery()`)

**Риски:** Нет.

---

## P3 — Низкий приоритет

### 10. Сохранённые запросы (закладки)

**Подход:** Расширение истории (п.3): добавить кнопку "Сохранить" → промпт с названием → хранение в отдельном ключе localStorage `saved_queries`. Отдельная секция в dropdown истории.

**Файлы:** `ui.html`

---

### 11. Подсветка замаскированных ячеек

**Подход:** При рендере таблицы добавлять CSS-класс `.cell-masked` для ячеек в замаскированных столбцах. Стиль: `background: var(--warning-soft)` с opacity 0.3.

**Файлы:** `ui.html` (CSS + JS в виртуальном скролле)

---

### 12. Toggle темы вместо двух кнопок

**Подход:** Заменить две кнопки (солнце/луна) на один toggle-switch. Клик переключает `theme-dark` ↔ `theme-light`.

**Файлы:** `ui.html` (HTML + CSS)

---

### 13. Звуковые и десктопные уведомления

**Подход:** 
- Звук: `new Audio(dataURI)` — короткий beep, закодированный в base64 прямо в JS.
- Desktop notification: `Notification.requestPermission()` + `new Notification()`.
- Чекбокс в настройках "Звуковые уведомления".

**Файлы:** `ui.html` (JS), возможно `web.go` (настройка)

---

### 14. Масштабирование таблицы

**Подход:** CSS-переменная `--table-font-size`, кнопки `A-`/`A+` меняют её. Сохранение в localStorage.

**Файлы:** `ui.html` (CSS + JS)

---

## Общие принципы

- Все состояния UI сохраняются в `localStorage` (тема, размер сплиттера, масштаб, история)
- Новый функционал не должен мешать существующему потоку MCP ↔ агент
- Каждый пункт тестируется изолированно: открыть UI → проверить функцию → убедиться что остальное не сломалось
