## Сборка приложения

Всегда собирать **без консольного окна** (GUI-режим Windows):

```bash
go build -ldflags "-H=windowsgui" -o trusted-gateway.exe .
```

Без `-H=windowsgui` при запуске появляется чёрное консольное окно, что неприемлемо для desktop-приложения.

## Кодировка UTF-8 в bash на Windows

Bash-среда Claude Code на Windows **не передаёт кириллицу** через stdout — символы превращаются в `????`. Это касается:
- вывода `python3` через pipe/heredoc
- `cat`/`head` файлов с UTF-8
- любых команд, которые выводят кириллический текст

**Решение**: при анализе данных из gateway (JSON с замаскированными полями) **не полагаться на кириллические имена ключей**. Вместо этого:
1. Идентифицировать ключи по формату значений (числа, псевдонимы с `_` и hex-суффиксом)
2. Работать с hex-суффиксами псевдонимов (`_59d1df0799`), которые ASCII-safe
3. Если нужна кириллица в выводе — писать результат в файл и читать через Read tool, а не через stdout

## GitHub — только через API

`git push` не работает (broken HTTPS helper, SSH pack fails). Все операции с GitHub — через `gh api` или `gh` CLI:

- **Пуш коммитов**: создать blobs → tree → commit → PATCH ref через `gh api`
- **Релизы**: `gh release upload <tag> <file> --repo alonehobo/1c-trusted-gateway`
- **Создание релиза**: `gh api repos/.../releases --input payload.json`
- Всегда использовать `--input <file>` для больших payload (ui.html, web.go не влезают в аргументы командной строки на Windows)

## Работа со шлюзом (тестирование, отладка)

Перед работой со шлюзом читать `README.md` (раздел TCP Bridge API) и `AGENT_INSTRUCTIONS.md` самостоятельно — не спрашивать у пользователя базовые вещи вроде порта или формата команд.

- TCP bridge: `127.0.0.1:8766`, протокол JSON → `shutdown(SHUT_WR)` → JSON
- Единственное что нужно спрашивать — bridge secret (генерируется при каждом запуске)
- Команды: `status`, `run_query`, `pull_note`, `apply_analysis`, `clear_session`
