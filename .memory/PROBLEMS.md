## Windows bash не передаёт UTF-8 кириллицу через stdout

**Симптом**: при выполнении Python-скрипта или `cat` через bash, кириллические символы в stdout заменяются на `??` / `\ufffd`. Файл при этом корректный (UTF-8), проблема только в pipe между процессом и терминалом.

**Пример**: JSON из gateway_query содержит ключи `Менеджер`, `Контрагент` — при чтении через bash они отображаются как `??????????`, из-за чего `KeyError` при обращении по имени.

**Обходной путь**:
- Определять ключи позиционно (`keys[0]`, `keys[1]`) или по формату значений (числа vs псевдонимы)
- Результат анализа записывать в файл, а не выводить в stdout
- Использовать Read tool для чтения файлов вместо `cat`
