## 2026-04-14 (фикс: PersistentTypePolicy терялся при сохранении настроек)
**Change ID:** 2026-04-14-type-aware-masking

- **Баг**: после `HandleSaveSettings`/`HandleImportSettings` создавался новый `app.Runtime = NewTrustedGatewayRuntime(newConfig)` с дефолтной `TypePolicy`. Пользовательский `PersistentTypePolicy` оставался на `app`, но **в рантайм не переприменялся**. Следствие: любое сохранение общих настроек (в т.ч. кнопка «💾 В настройки» у белого списка) молча стирало кастомные правила политики типов, пока не будет перезапуск приложения.
- **Фикс**: введён метод `reapplyPersistentTypePolicyLocked()` на `TrustedWebApp`. Вызывается после каждой замены `app.Runtime` в `HandleSaveSettings` и `HandleImportSettings`. Если `PersistentTypePolicy != ""`, мержит его поверх свежего default'а через `TypePolicy.MergePersisted`.
- Воспроизведение: добавили `Справочник.Номенклатура` → plain, нажали «Применить» — работает. Дальше нажали «💾 В настройки» у белого списка → runtime пересоздался → следующий MCP-запрос с ссылочной колонкой снова замаскирован. После фикса правило выживает.
- **UX**: в `tpAddRule` и `tpRemove` добавлено автосохранение (`tpSave(true)`). Теперь кнопки «+ Открыть»/«+ Маскировать»/«×» сразу применяют правило к рантайму. Кнопка «💾 Применить» остаётся как явный триггер (на всякий случай) и для batch-редактирования.

## 2026-04-14 (чистка дублей "Ссылка" в политике типов)
**Change ID:** 2026-04-14-type-aware-masking

- Из default'ов `type_policy.go` удалены формы со «Ссылка»: префиксы `ПеречислениеСсылка.`, `ПланСчетовСсылка.`, `ПланВидовХарактеристикСсылка.` и exact-типы `СправочникСсылка.Валюты/СтавкиНДС/СтраныМира/БанкиРФ`.
- Причина: MCP-сервис 1С (`mcp_ИнструментЗапросыКБазе/Ext/ManagerModule.bsl`, функция `СхемаКолонокТЗ`) формирует имена типов через `Метаданные.НайтиПоТипу(Тип).ПолноеИмя()` — это всегда форма **без «Ссылка»** (`Справочник.Контрагенты`, `Перечисление.ВидыДвижения`, `ПланСчетов.Хозрасчетный`). Подтверждено комментарием в самом BSL-исходнике (строка 224 ManagerModule.bsl) и ответом 1С-эксперта: `Метаданные.НайтиПоТипу().ПолноеИмя()` → без суффикса, `Строка(Тип)` / XDTO → со суффиксом.
- Формы со «Ссылка» в реальных JSON-ответах через наш MCP-сервис **никогда не появляются**, поэтому держать их в whitelist'е — мёртвый код, который только засоряет UI «Политика типов».
- Поправлены фикстуры тестов (`privacy_test.go:21-26`, `privacy_test.go:113`) — `СправочникСсылка.Контрагенты` заменена на `Справочник.Контрагенты`, что соответствует реальному формату.
- В `type_policy_test.go` кейсы для Ссылка-форм инвертированы: теперь они должны приводить к `Mask` (как любой неизвестный тип). Все тесты зелёные, бинарь собран.
- В файле `type_policy.go` оставлен комментарий со ссылкой на BSL-источник, чтобы при будущей правке не появилось искушение снова добавить «на всякий случай».

## 2026-04-14 (dot-эвристика для системных типов)
**Change ID:** 2026-04-14-type-aware-masking

- В `type_policy.go` инвертирована политика для **dotless** типов: всё без точки в имени → plain by default (это всегда платформенный примитив или системный enum). Маскируются только явные опасные типы из blacklist: `Строка`/`String`, `ХранилищеЗначения`/`ValueStorage`, `ДвоичныеДанные`/`BinaryData`.
- Проблема, которую закрыли: системные enum'ы платформы (`ВидДвиженияНакопления`, `ВидДвиженияБухгалтерии`, `ВидСчета` и т.п.) попадали под маску, потому что префикс whitelist'а `Перечисление.` их не ловит — они без точки в имени, не из метаданных конфигурации.
- Преимущество: не нужен растущий список системных enum'ов. Любой будущий платформенный тип автоматически открывается, blacklist стабилен.
- Приоритеты сохранены: `forced_mask_*` бьёт всё, пользовательский `plain_types` (например, добавить «Строка» вручную) бьёт blacklist.
- Добавлены тесты: `ВидДвиженияНакопления`/`ВидДвиженияБухгалтерии`/`ВидСчета`/`AccumulationRecordType` → plain; `ХранилищеЗначения`/`ValueStorage`/`ДвоичныеДанные`/`String` → mask; user override для blacklist; user forced-mask для dotless. Все тесты зелёные, бинарь собран.

## 2026-04-14 (Задачи 1 и 3 — реализация)
**Change ID:** 2026-04-14-type-aware-masking

- **Задача 1 — strip `ПРЕДСТАВЛЕНИЕ()`**: в `query_normalize.go` добавлены функции `stripPresentationCalls` и `findMatchingParen`. Банлист: `ПРЕДСТАВЛЕНИЕ`, `ПРЕДСТАВЛЕНИЕССЫЛКИ`, `PRESENTATION`, `REFPRESENTATION` (case-insensitive). Уважают вложенные скобки, строки `"..."` с `""`-экранированием и `//`-комментарии. До 8 проходов для вложенных вызовов. Подключено в `mcp_server.go:toolQuery` и `web.go:HandleQuery` перед `normalizeQueryAliases`. Добавлено 16 unit-тестов (`query_normalize_test.go`).
- **Задача 3 — type-aware маскирование**:
  - Клиент (`service.go`): при вызове `query` в аргументы добавляется `IncludeSchema: true`.
  - Парсер (`service.go`): новая функция `extractRowsWithSchema` парсит JSON-формат `{version, columns, rows}`; fallback на TSV. `buildColumnTypeMaps` индексирует схему для sanitizer'а.
  - Сессия (`service.go`): `TrustedSession.ColumnSchemas/ColumnTypes/ColumnTruncated` хранят схему. Передаются в sanitizer при каждом `SanitizeRows` и `remaskLocked`.
  - Type policy (`type_policy.go` + тесты): `TypePolicy.Decide(types, truncated) → plain/mask/unknown`. Дефолты: `Перечисление.*`, `ПланСчетов.*`, `ПланВидовХарактеристик.*`, примитивы (Число, Дата, Булево, УникальныйИдентификатор), несколько точных справочников (Валюты, СтавкиНДС, СтраныМира, БанкиРФ). Строка — mask по умолчанию. Truncated → всегда mask. Один опасный тип в составном → mask. `MergePersisted` сливает дефолты с user overrides. 17 тестов.
  - Sanitizer (`privacy.go`): новый приоритетный шаг `ds.typePolicy.Decide(...)` между force-mask и name-whitelist. При `TypeDecisionUnknown` логика падает на старые name-based правила (обратная совместимость). `privacy_test.go` покрывает интеграцию.
  - Persistence: `PersistentTypePolicy string` в `TrustedWebApp`; сохраняется/загружается в `settings.bin` через ключ `"type_policy"`. Новый хэндлер `POST /api/set_type_policy`, метод `HandleSetTypePolicy`. Загрузка в `main.go`.
  - UI (`ui.html`): новый sub-tab «Политика типов» в настройках. Таблица (Тип/Префикс, Политика plain/mask, удалить). Кнопки «+ Plain», «+ Mask», «Сбросить», «Применить». Префикс распознаётся по завершающей точке. Синхронизация из стейта идёт только когда таб не активен (чтобы не затирать работу пользователя).
- Все тесты (`query_normalize_test.go`, `type_policy_test.go`, `privacy_test.go`) — зелёные; `go vet` чист. Бинарь `trusted-gateway.exe` собран с `-ldflags -H=windowsgui`.
- Манaual smoke на реальной базе — остаётся. Если агент ещё не передаёт `IncludeSchema`, ответ от 1С придёт как TSV, и type-aware шаг просто пропустит колонки (поведение идентично старому).
