# Slash-команды проекта для агентов

## Назначение

Этот файл — единый словарь служебных slash-команд для `Codex`, `Cursor` и `Claude`.
Slash-команда — это явный управляющий сценарий, а не просто фраза пользователя.

## Правила резолва

1. Служебная команда начинается с `/`.
2. Совпадение должно быть полным. Не используй prefix-match, substring-match и "похожее" совпадение.
3. Если команда неизвестна, явно сообщай об этом пользователю.
4. Если команда распознана, сначала выполняй ее протокол, а уже потом переходи к обычной интерпретации запроса.

## Реестр команд

### `/prj-init`

- Назначение: инициализировать новый проект после копирования шаблона.
- Когда использовать: пользователь ввел ровно `/prj-init` или прямо просит инициализировать проект по шаблону.
- Источник правил: `docs/prj-init.md`.
- Ожидаемый результат:
  - заполнен `.memory/PROJECT.md`;
  - создан `Change ID` для стартовой спецификации проекта;
  - созданы `proposal.md`, `design.md`, `tasks.md`, `spec-delta.md` в `.memory/changes/<change-id>/`;
  - план и стартовая спецификация показаны пользователю на согласование до начала реализации;
  - в плане есть обязательный финальный блок с автотестами и передачей результата пользователю только после исправления ошибок или явной фиксации блокера.
