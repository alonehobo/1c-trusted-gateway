# Система алиасов-команд для агентов

## Назначение

Этот файл — единый источник истины для `%алиасов` в `Codex`, `Cursor` и `Claude`.
Алиас — это отдельная команда или контекст с явным поведением.

## Правила резолва

1. Алиас начинается с `%`.
2. При сравнении не учитывается регистр.
3. Имя алиаса разбивается на токены по `CamelCase`, пробелам, `_` и `-`.
4. Сравнение идет по полной последовательности нормализованных токенов.
5. Prefix-match, substring-match и "самое похожее" совпадение запрещены.
6. Если полного совпадения нет, агент обязан сообщить, что алиас неизвестен.
7. Если два алиаса после нормализации совпали, агент обязан сообщить о конфликте словаря.

## Практическая нормализация

- Для одиночных слов допускается приведение к базовой форме: `%Док` = `%Доки`.
- Для многословных команд базовая форма применяется к каждому токену отдельно.
- Общий корень не делает команды одинаковыми:
  - `%Доки` → `["док"]`
  - `%ДокиПроекта` → `["док", "проект"]`
  - `%ДокиAPI` → `["док", "api"]`

## Словарь алиасов

| Канонический алиас | Допустимые формы                          | Нормализованные токены  | Тип      | Целевой ресурс                                   | Предпочтительное действие                                                    |
| ------------------ | ----------------------------------------- | ----------------------- | -------- | ------------------------------------------------ | ---------------------------------------------------------------------------- |
| `%ШаблонПроект`    | `%Шаблон Проект`, `%Шаблон_Проект`        | `["шаблон", "проект"]`  | `path`   | `./`                                             | Использовать как ссылку на корень текущего шаблона проекта.                  |
| `%ПамятьПроекта`   | `%Память`, `%Память Проекта`              | `["память", "проект"]`  | `path`   | `.memory/`                                       | Использовать для текущего контекста, changelog, patterns и active task.     |
| `%ДокиПроекта`     | `%Доки`, `%Доки Проекта`, `%Документация` | `["док", "проект"]`     | `path`   | `docs/`                                          | Открывать локальную документацию и проектные протоколы.                      |
| `%ПравилаКурсора`  | `%ПравилоКурсора`, `%Правила Курсора`     | `["правило", "курсор"]` | `path`   | `.cursor/rules/`                                 | Искать локальные правила Cursor и их конкретные `.mdc`-файлы.                |
| `%MCPПроекта`      | `%MCP`, `%MCP Проекта`                    | `["mcp", "проект"]`     | `bundle` | `.mcp.json`, `.cursor/mcp.json`, `opencode.json` | При вопросах о подключенных MCP сначала проверять локальные конфиги проекта. |
| `%EnvПроекта`      | `%Env`, `%Окружение`, `%ПараметрыПроекта` | `["env", "проект"]`     | `path`   | `.dev.env`                                       | Читать локальные параметры проекта, если файл существует.                    |

## Поведение по умолчанию

- `%ПамятьПроекта` и `%ДокиПроекта` — разные команды.
- `%ШаблонПроект` и `%ДокиПроекта` — разные команды.
- `%MCPПроекта` и `%EnvПроекта` — разные команды.
- Если пользователь просит "добавить это в `%ШаблонПроект`", приоритет у корня проекта.
- Если пользователь просит "посмотреть правила в `%ПравилаКурсора`", открывай `.cursor/rules/`.

## Примеры интерпретации

- `Добавь это в %ШаблонПроект` → работать с корнем проекта.
- `Покажи, что лежит в %ПамятьПроекта` → открыть `.memory/`.
- `Посмотри правила в %ПравилаКурсора` → открыть `.cursor/rules/`.
- `Проверь %MCPПроекта` → посмотреть локальные конфиги MCP.
